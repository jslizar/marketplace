# AI-tells — the canonical list

THE single source of truth for AI-tells across every engine. The drafters (`copy-post` via `references/deslop-pass.md`, `abm-draft`, `funnel-draft`) rewrite against this list during drafting; `post-qa` flags against it as the final backstop. Don't maintain a second list anywhere — edit this one.

Goal: the post reads like the Author wrote it, not a model. Preserve anything the Style Spec or the client's Voice section marks as an intentional signature move (e.g. deliberate staccato).

## Kill on sight — no exceptions

1. **Em-dashes.** Every one — no `—`, no ` - ` standing in for one, no `--`. Replace with a comma, a period, parentheses, or restructure. (This is the strict rule; a single em-dash is still a tell.)
2. **"Not X, it's Y" contrasts** and every cousin: "isn't X, it's Y", "not because X, but Y", "it's not about X, it's about Y", "not just X but Y", "stops being X and starts being Y". State Y directly and drop the negation.
3. **Throat-clearing openers** — "In today's fast-paced world", "Let's be honest", "Here's the thing", "The truth is". Lead with the hook.
4. **Exclamation marks.**

## Cut

- **Hollow adverbs / intensifiers** — really, just, genuinely, honestly, simply, actually, literally, truly, incredibly, "game-changing", "powerful". Delete or replace with a concrete detail. Adverb stacks doubly so.
- **Emphasis crutches** — "Let that sink in", "Make no mistake".
- **Rule-of-three padding** — "faster, smarter, better." Keep one if it's real; drop the filler. Two items beat three.
- **Business jargon** — leverage → use, navigate → handle, unpack → explain, deep dive / "dive in" → analysis / the point itself, lean into → accept, double down → commit, circle back → revisit, moving forward → next.
- **Rhetorical setups and meta-commentary** — "What if I told you", "Think about it", "Let me walk you through". Rhetorical-question hooks count, unless the format spec's hook pattern IS a question.
- **Negative listing** — "Not a X. Not a Y. A Z."
- **Aphoristic closers / pull-quotes** — if the last line sounds like it wants to be framed, rewrite it as a plain statement.

## Rewrite

- **False agency / passive voice** — no inanimate object doing a human verb ("the data tells us", "the complaint becomes a fix"). Name the actor, or use "you".
- **Vague declaratives** — "the implications are significant", "the reasons are structural". Name the specific thing. No lazy extremes (every, always, never) doing vague work.
- **Vague authority claims** — "studies show", "everyone knows". Cite or cut.
- **Narrator-from-a-distance** — "Nobody designed this", "People often find". Put the reader in the room: "you" beats "people".
- **One-word-per-line "broetry"** used for drama with no substance.
- **Hashtag stuffing** — more than 0–3 targeted tags.
- **Emoji bullets** as a default — only if the Style Spec / Voice says the Author uses them.
- **Fake vulnerability** ("I almost gave up…") the context doesn't support.
- **Metronomic rhythm** — three consecutive sentences of matching length; every paragraph ending on a punchy one-liner. Vary it.

## Smell test

- Could this sentence appear on any company's post? If yes, make it specific to the client or cut it.
- Does it *sound* like the Author reading it aloud? If not, plain it down.
- Is every number/name real (from context Canon/Proof or this run's sourced research)? If not, that's a hard block, not a flag (see `post-qa/SKILL.md`).

## Don't over-correct

- Preserve anything the Style Spec marks as an intentional signature move: deliberate staccato, punchy fragments, the Author's closers, formatting conventions.
- Keep the Author's real voice, contractions, and rhythm. Don't flatten the post into corporate prose.
- The line: cut **AI tells**; keep **the Author's deliberate craft**. When a fragment is doing rhythmic work in the matched style, it stays. When fragmentation is manufactured profundity, it goes.
