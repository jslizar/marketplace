# Tool workflow demo

- **Archetype**: "let me show you how I use {tool} for {job}" — a real working system walked through step by step, with results and a gate
- **Funnel stage**: MOFU — makes the approach credible by showing the actual machine, not claims about it.
- **Stage mechanic**: The ICP is actively evaluating this tooling; a practitioner's real architecture ("11 APIs and 72 automation scripts", "$25k in extra revenue in 30 days") is exactly what they want to steal. Massive comment-gate conversion (1,964 comments).
- **Platform**: LinkedIn
- **Source examples**: curated from Millie's bank; refreshed only when the bank updates.

  - https://www.linkedin.com/posts/jonathan-peslar_i-used-claude-cowork-for-my-seo-for-30-days-share-7428114528657272832-uUly — John Peslar, 294 reactions / 923 comments, 2026-02-17
- **Hook pattern**: "Let me show you how to use {tool} for {job}. It's like having {enormous equivalent} working for you." / "I used {tool} for my {job} for {n} days. The result: {number}."
- **Skeleton** (block sequence):
  1. Demo-invitation or experiment-result hook with one arresting equivalence/number
  2. System credibility line: the scale of what's running (APIs, scripts, prompts)
  3. The old-way/new-way contrast (why the default approach fails)
  4. The walkthrough: numbered stages, each with what the tool actually does (named steps like "The Recon", "The Signal")
  5. Honest mechanics note (what it adapts to, what it can't do)
  6. Gate or follow close: comment keyword for the system/prompts
- **Rhythm & formatting**: Numbered stages with named steps, short imperative lines, 300–500 words.
- **Why it works**: Working systems beat opinions — naming the internal steps makes it feel open-sourced. The experiment variant (30 days, $25k) adds a falsifiable frame that reads as honest.
- **Adaptation notes**: Must change: the tool, the system, the numbers (the client must actually run this system). Must NOT change: named-steps walkthrough — an unnamed list of "steps" loses the borrowed-artifact feel.
- **Watch-outs**: The system must survive scrutiny from practitioners in the comments. Result numbers need provenance (Proof section). If the client is the tool's vendor, disclose it — undisclosed vendor demos get flamed.
