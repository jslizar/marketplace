# Anti-process hiring manifesto

- **Archetype**: "we killed {sacred hiring ritual}" + what we do instead, doubling as a job ad
- **Funnel stage**: TOFU — earns reach through hiring-process heresy; the role fills itself from the comments.
- **Stage mechanic**: Attacks a ritual the ICP privately resents (LeetCode, whiteboards, credential screens). Engineers/operators share it in vindication; candidates self-select in the comments. The concrete "what we do instead" makes it substance, not attitude.
- **Platform**: LinkedIn
- **Source examples**: curated from Millie's bank; refreshed only when the bank updates.
  - https://www.linkedin.com/posts/sugju-choi_we-killed-leetcode-at-virio-no-online-assessments-activity-7397686902839013376-BKpD — Sugi Choi (Virio), 1,995 reactions / 292 comments, 2025-12-04
  - https://www.linkedin.com/posts/ericlay-virio_were-hiring-a-head-of-sales-development-activity-7391512082879258626-RMOG — Eric Lay (Virio), 426 reactions / 112 comments, 2025-11-04 (comp-forward role-post variant: "$250k–$310k TC" in line two)
- **Hook pattern**: "We killed {ritual} at {company}. No {artifact}. No {artifact}. No \"{cliché test}\" questions." — or comp-forward variant: "We're hiring a {role}. And we're paying ${range} to find our next killer."
- **Skeleton** (block sequence):
  1. The kill statement + staccato list of banished rituals
  2. Why the ritual is now meaningless (a real shift — e.g., AI assistants break assessments)
  3. What we actually select for, stated as identity ("we're building products, not algorithms")
  4. The replacement process, numbered and concrete (projects shown, reference calls)
  5. "That's it." beat — the brevity is the point
  6. Self-selection close: "if you can {bar}, we should talk" + comment prompt
- **Rhythm & formatting**: Staccato negations up top, numbered process block, 150–350 words. Emoji-numbered lists acceptable in this genre.
- **Why it works**: Ritual heresy earns the reach; the disclosed process earns the trust. Comp transparency (in the role-post variant) is its own news hook.
- **Adaptation notes**: Must change: the ritual, the replacement process, comp numbers (all true and Canon-checkable). Must NOT change: kill-list hook shape and the "that's it" brevity beat.
- **Watch-outs**: If the client's real process still includes the ritual they're mocking, this post is a liability. Comp ranges are commitments — verify before publishing.

## Top example — full post text

We killed LeetCode at Virio.

No online assessments. No whiteboard problems. No "reverse a binary tree" questions. With AI coding assistants everywhere, technical rounds are borderline meaningless now. Anyone can pass them with the right setup of ChatGPT + Claude + Cluely.

We're not building algorithms at Virio. We're building a product that actual customers pay for and use every day.

I don't need someone who's solved 500 LeetCode hard problems. I need someone who can ship a feature from idea → design → code → production in a week.

This is what we do instead to find real builders:

1️⃣ Previous projects: Show us what you've actually built and shipped.
2️⃣ Reference calls with your past teammates: We want to hear how you actually performed on a team, not how well you prepped for an interview.

That's it.

If you can build, iterate fast, and care about the end user experience more than the perfect algorithm - we should talk.

Drop a comment here if you're interested, let's build real products together.
