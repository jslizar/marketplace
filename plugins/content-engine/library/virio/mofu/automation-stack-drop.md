# Automation stack drop

- **Archetype**: "We replaced {expensive thing} with {system}" + the system itself, comment-gated
- **Funnel stage**: MOFU — makes the problem urgent and the approach credible.
- **Stage mechanic**: The replacement claim ($300K team → 7 agents) is a cost-anxiety hook aimed straight at evaluators; the name-dropped adopter tiers (VCs, 8-figure brands) provide social proof; the gate captures everyone who wants the shortcut.
- **Platform**: LinkedIn
- **Source examples**: exactly 2, refreshed by the scheduled library-refresh task from the viral_posts_all corpus.

  - https://www.linkedin.com/posts/cas-feijen-66634426b_these-7-ai-marketing-agents-made-us-256983-activity-7345442234810458112-0Kzj — Cas Feijen (MakeUGC), 794 reactions / 2,415 comments, 2025-06-30
- **Hook pattern**: "We replaced a {$X/year cost center} with {n} {new-tech units}"
- **Skeleton** (block sequence):
  1. Replacement-claim hook with the dollar figure
  2. One-line system description (what the units do end-to-end)
  3. Social-proof tiers: "I've shared this with top {ICP} from:" + dash list of impressive categories
  4. Adoption line ("Every one of them started using it")
  5. "Here's how it works:" — 2-step mechanism, concrete
  6. The give: "I'm giving you access to the full stack"
  7. Gate: comment "{keyword}" (+ connected) + P.S. repost-for-early-access
- **Rhythm & formatting**: 120–170 words. Dash lists. Dollar figures and counts throughout. Direct, no adjectives where a number fits.
- **Why it works**: Replacement math is the most clickable MOFU claim in any downturn; the tiered adopter list lets readers locate themselves ("if YC startups use it…"). Double CTA (comment + repost) compounds distribution.
- **Adaptation notes**: The replaced cost and the system must be real and demonstrable — this format gets stress-tested in comments. Adopter tiers must be truthful categories.
- **Watch-outs**: "Replaced a team" framing can read as celebrating layoffs — some ICPs (HR!) will turn on it; reframe as "replaced the spend" where sensitive. Overclaimed automation gets exposed fast.
