# ICP service roundup

- **Archetype**: Curated, no-strings list serving the ICP's immediate need
- **Funnel stage**: MOFU — makes the problem urgent and the approach credible.
- **Stage mechanic**: Doing the ICP's legwork for free (jobs, tools, grants, events — whatever they're hunting) earns saves, follows, and standing as the person plugged into their world. Credibility via service, not claims.
- **Platform**: LinkedIn
- **Source examples**: exactly 2, refreshed by the scheduled library-refresh task from the viral_posts_all corpus.
  - https://www.linkedin.com/posts/michellejager_remoteopportunity-hiring-remotework-activity-7145784721476579328-fTcb — Michelle Jager, 2,355 reactions / 249 comments, 2023-12-27
  - https://www.linkedin.com/posts/kianakazemi2025_climatejobs-techforgood-sustainabilitycareers-activity-7352012910472163328-8Bn2 — Kiana Kazemi, 948 reactions / 30 comments, 2025-07-18
- **Hook pattern**: "Looking for {the thing the ICP is hunting}? Check out these {n} opportunities:"
- **Skeleton** (block sequence):
  1. Direct-address question hook naming the hunt
  2. The list: dash bullets, each `{item} at {named org}` — 10–15 entries, all real and current
  3. Disclosure line (shared to help; not affiliated)
  4. Well-wish + hashtags
- **Rhythm & formatting**: The list IS the post. One item per line. 100–170 words. No gate, no CTA beyond the list itself.
- **Why it works**: Pure utility with zero ask reads as generosity; taggable named orgs extend reach (companies and candidates both engage). The disclosure line converts "promoter" into "community member".
- **Adaptation notes**: Curate the thing the client's ICP hunts weekly (for a devtools client: "10 OSS projects paying maintainers"; for HR tech: exactly this example). Every entry verified live this run — dead links are the format's death.
- **Watch-outs**: Stale or padded entries get called out and invert the trust play. Sneaking the client's own product into the list reads as bait-and-switch — if it belongs, disclose it.
