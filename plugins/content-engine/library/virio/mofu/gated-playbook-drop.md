# Gated playbook drop

- **Archetype**: Finished-artifact giveaway, gated on a comment keyword
- **Funnel stage**: MOFU — makes the problem urgent and the approach credible.
- **Stage mechanic**: The comment gate converts credibility into a captured, warm list — every commenter self-identifies as an evaluator of exactly this problem. The proof arrows justify why the artifact is worth the ask.
- **Platform**: LinkedIn
- **Source examples**: exactly 2, refreshed by the scheduled library-refresh task from the viral_posts_all corpus.
  - https://www.linkedin.com/posts/mattlakajev_after-3-years-i-just-finished-the-linkedin-activity-7341020247585411073-KIpm — Matt Lakajev, 1,726 reactions / 9,995 comments, 2025-06-18
  - https://www.linkedin.com/posts/jessievanbreugel_i-should-be-selling-this-but-im-handing-activity-7358108240204779523-yYUZ — Jessie van Breugel, 848 reactions / 3,096 comments, 2025-08-04
- **Hook pattern**: "After {timespan}, I just finished — {artifact name}. {size stat}. You can have it for FREE."
- **Skeleton** (block sequence):
  1. Completion hook: timespan + artifact + FREE
  2. First gate line: 'Comment "{KEYWORD}" and I'll send it to you'
  3. Proof arrows (→): 4–5 quantified results the system behind the artifact produced
  4. Curiosity arrows (→): 2–3 never-shared specifics inside the artifact
  5. Outcome promise, one line, plain
  6. Gate repeated verbatim
  7. P.S. with an edge ("warning: this is the red pill")
- **Rhythm & formatting**: Arrow bullets for both proof and contents. Numbers everywhere. 130–180 words. The gate line appears exactly twice.
- **Why it works**: Reciprocity (real artifact, free) + specificity (14,475 words; $3.32M closed) + the comment gate turning interest into algorithm fuel — every "TRUST" comment boosts distribution AND builds the DM list.
- **Adaptation notes**: The artifact must exist and be genuinely good — this format is a promise. Proof numbers come from the client's Proof section only. Keyword should echo the artifact's theme.
- **Watch-outs**: Weak artifact = one-time trick that burns trust. Comment-gating without actually DMing every commenter is reputation suicide. Some audiences read comment gates as spammy — check the client's voice rules first.
