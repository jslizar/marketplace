# Drafting framework

Three phases: **hook → body → CTA**.

## Phase 1 — Hook

Build per `hook-framework.md`: the hook type and pattern come from the matched spec; each of the 3 candidates takes a different entry point into that same pattern. Keep the first line under the mobile fold.

## Phase 2 — Body

Deliver on the hook's promise. Shape it three ways at once:

- **Fill the template.** Work block by block through the `post-deconstruct` template, honoring each block's constraint (char ceiling, glyph, sentence length). Use the `post-structure` blueprint for the skeleton, transitions, whitespace map, and close — copy the *shape*, not the example's words.
- **Serve the funnel stage** (from the stage directive):
  - *TOFU* — a relatable story or observation; land one clear takeaway; no selling.
  - *MOFU* — teach a method or framework; soft proof (how the Author thinks); hint at the offer.
  - *BOFU* — a specific result or named offer; proof in the proof slot; build toward the ask.
- **Carry the client's substance.** Topic on a Pillar, claims only from context/research, named accounts/signals only from `icp-research`. Speak as the **Author**.

**Proof slot rule:** if context Proof is "None on record", fill the proof slot with credibility/process (first-hand experience, the specific diagnosis, the method) — never an invented metric or case.

## Phase 3 — CTA

Set strength by stage (from the directive):

- *TOFU* — none, or a soft discussion prompt ("curious how others see this").
- *MOFU* — soft ("follow for more", "DM me 'X'").
- *BOFU* — the client's **Conversion ask**, stated plainly.

If a link is needed, put it in the **first comment** and write that comment; keep the post body link-free for reach.

## ABM mode

When `icp-research` supplies a target set, choose with the user:

- **Segment post** — one post that speaks to the *pattern* across the accounts (name the shared trigger/pain, not one company).
- **Batch** — one account-aware post per target: weave that account's signal/angle into the body, keep the Author's voice, and match CTA strength to how warm the account is (don't hard-pitch a cold account).

Tag each batch post with the company and the angle used.

## Before presenting

Run every candidate through `style-match` first (fidelity score; apply its fixes and re-draft if below threshold), then `post-qa` (brand voice, Canon, Proof, AI-tells). Show 3 candidates, each with its **match score** + hook char count + stage + a one-line differentiator, then a recommended pick and why.
