# Feedback log

The banned list exists because of feedback. This file keeps that loop alive: a running record of what actually went out and what it did, so the registers and rules improve over time instead of freezing.

## How to use it

- **Before drafting** for a repeat target, a repeat client, or a familiar pattern, skim the log for anything relevant. If a phrasing flopped before, don't repeat it.
- **After you learn an outcome** (sent / reply / no reply / call booked / a "don't say that again"), append a row.
- **When a clear pattern emerges** (a phrasing that reliably lands or reliably dies), promote it: add the winner to the registers in `examples.md`, or add the loser to `references/banned-phrases.md`. Note the promotion in the row.

Keep entries short. The point is signal, not a CRM.
